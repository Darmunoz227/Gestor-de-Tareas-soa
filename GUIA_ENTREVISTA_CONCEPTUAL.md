# Guía para Entrevista Conceptual

## Introducción
Esta guía está diseñada para ayudar a los candidatos a prepararse de manera efectiva para entrevistas técnicas, brindando explicaciones conceptuales fundamentales que se espera que conozcan.

## Estructura de Datos
* **Listas**: Una colección de elementos ordenados.
  * Se pueden modificar (listas dinámicas). 
* **Conjuntos**: Una colección de elementos únicos y no ordenados.
* **Diccionarios (Hashmaps)**: Un tipo de estructura de datos que almacena pares clave-valor, proporcionando una búsqueda rápida.

## Algoritmos
* **Ordenamiento**: Proceso de reorganizar una lista o conjunto de datos.
  * Ejemplos comunes incluyen Quick Sort, Merge Sort, y Bubble Sort.
* **Búsqueda**: Técnicas para encontrar elementos en estructuras de datos.
  * Búsqueda binaria en listas ordenadas.

## Paradigmas de Programación
* **Programación Orientada a Objetos (POO)**: Un enfoque que utiliza "objetos" para modelar datos.
  * Conceptos clave: clases, objetos, herencia, y encapsulamiento.
* **Programación Funcional**: Un enfoque que evita el uso de estados mutables y se basa en funciones puras.

## Bases de Datos
* **SQL vs NoSQL**: Diferencias entre bases de datos relacionales y no relacionales.
* **Consultas**: Cómo realizar operaciones de lectura y escritura en las bases de datos.

## Redes
* **Modelo OSI**: Un marco para entender cómo operan las redes de computadoras.
  * Capas del modelo: física, enlace de datos, red, transporte, sesión, presentación, y aplicación.
* **Protocolos**: Comprensión básica de protocolos comunes como HTTP, FTP, y TCP/IP.

## Conclusión
Estar preparado para una entrevista técnica es fundamental. Conocer estos conceptos ayudará a los candidatos a comunicar sus ideas de manera efectiva y a demostrar su comprensión de temas técnicos.
